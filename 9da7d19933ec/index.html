GitHub + Hexo 搭建个人博客详细教程

[TOC]

1、前言

用户搭建一个属于自己的个人网站，分享信息更加便捷。本文详细记录了搭建步骤，享受从 0 到 1 的过程。由于技术更新较快，很多同类文档已经失去时效性，本文基于当前最新版本重新搭建。

2、技术选择

通常来说，建立博客的通常渠道包括以下 3 种：

  • 博客平台注册，如：博客园、CSDN、新浪博客、知乎、简书、掘金等
  • 博客框架搭建，如：WordPress、Jekyll、Hexo 等
  • 重新造轮子，代码实现

其中，第一种最简单，也最受限，容易被删帖删号禁言；第二种稍复杂，但可定制化较高，是很多人的选择；最后一种，高情商表达是重复造轮子，锻炼编程能力，低情商表达是没有专注博客写作，本末倒置。本文选择的是第二种：既不受限，难度也不大。

为什么选择 GitHub 和 Hexo？

  • 纯静态博客，部署简单、成本低廉、运行优化非常高效。
  • Hexo 静态博客框架可以在云端，也可以在本地电脑上甚至手机上。
  • Hexo 框架的博客网站没有网站后台，不存在后台安全漏洞的问题。
  • 因为是纯静态的，几乎所有的 CDN 服务都可以完美的加速提升浏览速度。
  • 搜索引擎对纯静态网站有天然的青睐度，收录、权重优先权明显。
  • 纯静态网站会让网络攻击成本大大的增加，也就是说纯静态网站被攻击的可能性更低了。

3、工具介绍

3.1 Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

3.2 GitHub

GitHub 是一个代码托管网站和社交编程网站。

3.3 GitHub Pages

GitHub Pages 是用来托管 GitHub 上静态网页的免费站点。

简单介绍就是：利用 Hexo（word）在个人电脑上生成博客文档，将其传输到 Github（微博）托管代码，再由 Github（微博）提供的 Github Pages（微博文章）服务将博客部署到网上，这样你就通过手机或者电脑浏览博客。
